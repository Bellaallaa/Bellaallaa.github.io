常用命令

命令 作用
hexo init 初始化一个新的博客
hexo new "My New Post" 新建一篇博客
hexo n 同上
hexo clean 清理缓存文件与已生成的静态文件(常用语hexo g 前)
hexo generate 生成静态网页(在public/文件夹中)
hexo g 同上
hexo server 启动本地服务器, 默认端口4000, 可预览博客
hexo s 同上
hexo deploy 部署网站到远程
hexo d 同上
hexo list [type] 列出某类内容(如posts, pages, tags 等)
hexo version 查看hexo的版本号
hexo help 产看命令帮助文档

一些注意…

  1. 在部署时, 有时会因为网络问题出现报错, 可以过段时间再次尝试
  2. 使用hexo d命令部署后需要等待一段时间才能在网页看到更新后的显示, 可以使用ping github.com进行测试
  3. 关于图片显示
    1. 我们在本地将图片复制粘贴到VSCode中时能正常显示, 但是上传到服务器后可能显示
    2. 这是因为本地的VSCode的Markdown预览(以及Hexo本地服务器)能直接访问你的电脑路径。而GitHub服务器上没有那些图片文件，所以显示图片丢失
    3. 解决方法：将_config.yml文件中post_asset_folder变量(我的在第43行)的值改为true，再进行hexo new "文章标题"等一系列操作
    4. 此时会自动生成
    5. 此时直接把图片粘贴到markdown文档中，会自动生成
