Git 常用命令

Created2025-10-30|Updated2025-10-30
|Post Views:

Git 是最常用的分布式版本控制系统之一，命令非常多，但其实常用的就几十个左右。下面是一份「最常用 Git 命令速查表」，并按用途分类

🧱 一、初始化与配置

命令 说明
git init 在当前目录初始化一个新的 Git 仓库
git clone <url> 克隆远程仓库到本地
git config --global user.name "YourName" 设置全局用户名
git config --global user.email "you@example.com" 设置全局邮箱
git config --list 查看当前配置

📁 二、查看仓库状态与历史

命令 说明
git status 查看当前工作区状态（修改、暂存等）
git log 查看提交历史
git log --oneline --graph --all 以简洁图形方式查看提交历史
git show <commit> 显示某次提交的详细信息
git diff 查看工作区与暂存区的区别
git diff --staged 查看暂存区与上一次提交的区别

💾 三、添加与提交修改

命令 说明
git add <file> 添加单个文件到暂存区
git add . 添加所有修改到暂存区
git commit -m "message" 提交修改并添加提交说明
git commit -am "message" 跳过暂存区直接提交已追踪文件的修改
git restore <file> 撤销工作区修改
git restore --staged <file> 将暂存区文件移回工作区（取消暂存）

🌳 四、分支管理

命令 说明
git branch 查看本地分支
git branch <name> 新建分支
git checkout <name> 切换到某分支
git checkout -b <name> 新建并切换到该分支
git merge <name> 将指定分支合并到当前分支
git branch -d <name> 删除本地分支
git switch <name> 切换分支（新版推荐）
git switch -c <name> 新建并切换分支（新版推荐）

🌐 五、远程仓库操作

命令 说明
git remote -v 查看当前远程仓库
git remote add origin <url> 关联远程仓库
git fetch 获取远程更新但不合并
git pull 拉取远程更新并合并
git push 推送本地更新到远程
git push -u origin <branch> 首次推送并设置默认远程分支
git push origin --delete <branch> 删除远程分支

🔙 六、回退与撤销操作

命令 说明
git reset <file> 取消暂存某文件
git reset --hard <commit> 回退到指定提交（会丢失修改）
git reset --soft <commit> 回退到指定提交但保留修改
git revert <commit> 生成一个新的提交来撤销某次提交
git reflog 查看所有操作日志（可找回丢失的提交）

🧩 七、标签（Tag）

命令 说明
git tag 查看标签
git tag <name> 创建轻量标签
git tag -a <name> -m "message" 创建带注释标签
git push origin <tag> 推送某个标签到远程
git push origin --tags 推送所有标签到远程

🧰 八、其他实用命令

命令 说明
git stash 临时保存当前修改
git stash pop 恢复最近一次保存的修改
git cherry-pick <commit> 将某次提交应用到当前分支
git blame <file> 查看文件每一行的修改者和提交号
git clean -fd 清除未跟踪的文件和文件夹
